Practice writing methods that use loops to process an array of Car objects.

Note:  You do NOT need to turn in this Lab 10 NetBeans project to the D2L dropbox, unless you were absent on the day we worked on it in class, or were not actually working on this lab activity during the class period when it was assigned.  You will probably want to come back and finish the entire lab later as preparation for the upcoming Exam.

  2. Examine at the Car and the MUFrame class definitions.

    1. You do not need to change the Car class definition at all, but take a look at it now in the Netbeans editor window so you can see the methods that "get" the Car data members.  You will need to call these "getter" methods to complete this lab activity.

    2. Now take a look at the MUFrame class definition in the Netbeans editor.  Notice that this program declares two different arrays of Car objects, and instantiates them with a different number of elements.  Then each position in both arrays is instantiated with specific values.

      The program then calls a method twice to display the speed of the fastest car in each array, and then skips a blank line.  Then the program calls a different method twice to return the speed of the fastest car in each array, and skips a blank line.  The methods themselves follow – you can use these as examples when solving the problems given in this lab.

  4. Create and Test Several Additional Methods that will process an array of Car objects.

    1. You will work on this part of the lab activity in stages ... one method definition and method call combination at a time.  Use the sequence described in the previous instruction set for the displayAllCars method for each of the following methods.

      There are two styles of methods for you to write in this lab activity.  Methods whose name starts with "display" are supposed to use System.out.println statements to display their results directly and immediately in the Java Console.  Methods whose name starts with "return" should NOT include any System.out.println statements, but should return their computed answer to the calling point via the use of a return statement.  When a "return"-style method is called from the MUFrame constructor, it must be called from within a System.out.println statement.

      Each of these methods requires that an array of Car objects be passed into the method as a parameter. You can write the methods in any order you like, but get one working before starting on another. Although you may not have time to finish all of the methods listed here, complete as many as you can in the schedule lab time today, and finish the rest whenever you can.

    2. If your answer is something like 25 or 25.0, you need to do something to not lose the fractional part of the average.

    3. Write and call a method named returnWorstMPGFor which will return the worst (i.e., lowest) miles per gallon (mpg) for any Car object in the array.

    4.       Call this method in MUFrame twice, once for each array of Car objects, and be sure to display a blank line after each of the two listings.
