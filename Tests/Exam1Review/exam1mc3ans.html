Multiple Choice Question # 3 Answer:

Consider the following class definition that defines a private data member called theRadius: 
    public class someClass {
        private Double theRadius;

        // .... rest of stuff would go here
		
    } // end of class someClass
What is wrong with the following Java method definition that would be placed in someClass? 
    public void setTheRadius( Double rad ) {
        rad = theRadius;
    } // end of setTheRadius
method heading should be   public Double setTheRadius( ) {
method heading should be   public Double setTheRadius( Double rad ) {
assignment statement should be   theRadius = rad;
Nothing is wrong with this method definition
The purpose of a "setter" method is to store the parameter value in a private data member.  With an assignment statement, the value to be stored is placed on the right-hand side of the assignment operator, and the destination where the value is to be stored goes on the left-hand side.  So rad = theRadius; does not store a value in theRadius, because the name of the private data member is not on the left-hand side of the =.
