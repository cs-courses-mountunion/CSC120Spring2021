  4. Write a complete class definition including private data, constructor, getters, setters, toString, and draw.

    1. Create a new Netbeans project called Exam1d. Make sure the Location is in your H:\CSC120\Java folder.

    2. Minimize NetBeans, but do not close the project or NetBeans.

    3. Copy needed files into your src folder.

      • In a web browser right click on
        MUPanel.java,
        and choose Save Target As... and save the files in the src folder of your project.

    4. Look over the MUPanel.java file.

    5. Then write a complete class definition for a class called ThingToBeDrawn, that is used by this program.  Right-click on the src folder of the Exam1d project and choose "New | Java Class" to create a file named ThingToBeDrawn.java for this part of the lab.

    6. The private data (properties) for the ThingToBeDrawn class should be an Integer called anchorX, an Integer called anchorY, a Color named theColor, and a String called nameOfThing.

    7. You will need to add an import statement at the top of the ThingToBeDrawn class, so that it will find the Color class and the Graphics class. At the top of the ThingToBeDrawn.java file above the class header line add this line: 
              import java.awt.*;

    8. The constructor for the ThingToBeDrawn class should have 4 parameters (2 Integer values, one Color and one String), and use the parameters to initialize the corresponding data members.

    9. Now write the getAnchorX (returns the value of the anchorX data member) and setAnchorX (sets the anchorX data member to the value of the method's one parameter) methods for the ThingToBeDrawn class.

    10. Write a draw method in the ThingToBeDrawn class. This method returns nothing (void), and expects a Graphics parameter called g.  First of all, this method should call the g.setColor method to change the drawing color to theColor.  Next, fill a rectangle that is located at the (anchorX, anchorY) point on the screen – make the rectangle 80 pixels wide and 50 pixels high.  Then it should use g.drawString to draw nameOfThing below the rectangle on the screen.

    11. Run the program, fix any errors, etc to test it. After you get this working, proceed to the next step.

    12. Edit your ThingToBeDrawn class adding a toString method.

      The toString method for a class should return a String that describes the object in terms of its data values. Your ThingToBeDrawn class has data members anchorX, anchorY, theColor, and thingName, so your toString method should return a String that uses all of those variables in a sentence. If you aren't sure how to make the toString method, then look at the toString method for Exam1b's SportsCar class as an example.
