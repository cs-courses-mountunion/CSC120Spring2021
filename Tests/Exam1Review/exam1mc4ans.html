Multiple Choice Question # 4 Answer:

Consider the following class definition that defines a private data member called gramsOfProtein: 
    public class someClass {
        private Integer gramsOfProtein;

        // .... rest of stuff would go here
		
    } // end of class someClass
What is wrong with the following Java method definition that would be placed in someClass? 
    public void getGramsOfProtein( ) {
        return gramsOfProtein;
    } // end of getGramsOfProtein
method heading should be   public Integer getGramsOfProtein( ) {
method heading should be   public void getgetGramsOfProtein( Integer gProt ) {
body of method should contain the statement   gramsOfProtein = gProt;
Nothing is wrong with this method definition
The purpose of a "getter" method is retrieve the value of a private data member from an object.  In order to return a value from a method to the caller of the method, the Java system must know the data type of the value to be returned.  Therefore, a "getter" method cannot have a void return type — the return type of the method must be the same as the data type of the private data member that is being returned.
